🏗️ 项目架构

整体架构设计

Vtox系统采用分布式微服务架构，具有高度的可扩展性和容错能力。系统主要由以下几个核心层次组成：

数据采集层 → API网关层 → 消息队列层 → 分布式处理层 → 数据存储层 → 前端展示层

🏗️ 代码工程架构图

微服务拆分方案

🌐 API Gateway Service (端口8000)

  • 统一入口和路由管理
  • 负载均衡和服务发现
  • 认证授权和API限流
  • CORS跨域处理

📊 Diagnosis Coordinator (端口8001)

  • 分布式诊断系统协调
  • Consumer Group管理
  • 跨节点负载均衡
  • 智能任务分发

🔧 Fault Analysis Workers (8002-8010)

  • Worker-1: 匝间短路 + 绝缘故障
  • Worker-2: 轴承 + 偏心故障
  • Worker-3: 断条故障
  • 支持水平扩展到50+节点

🔗 Redis Stream Bridge

  • 实时数据流处理
  • WebSocket消息转发
  • 高性能批处理优化
  • 自动故障转移

数据流架构

车辆模拟器(50辆车) ↓ HTTP POST API网关(FastAPI) ↓ 写入数据流 Redis Streams(motor_raw_data) ↓ 并行分发 分布式消费者(10个处理单元) ↓ 故障诊断算法 Redis Streams(fault_diagnosis_results) ↓ 结果聚合 结果聚合器 ↓ WebSocket推送 前端应用(Vue.js)

🚀 优化建议

🔧 技术架构优化

🏗️ 架构层面优化

  • 引入服务网格 - 使用Istio或Linkerd简化微服务通信
