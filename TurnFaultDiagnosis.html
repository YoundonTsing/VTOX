3. 匝间短路故障特征

电流不平衡

匝间短路导致受影响相的电流幅值增大，造成三相电流不平衡。通过计算三相电流RMS值的偏差来检测。

三次谐波增加

匝间短路会导致电机磁场分布不均匀，产生额外的三次谐波分量。通过频谱分析检测三次谐波与基波的比值。

负序分量增加

匝间短路会产生额外的负序电流分量，破坏三相平衡。通过对称分量分析计算负序分量占比。

4. 系统组件详细设计

4.1 数据源模拟应用

数据源模拟应用负责生成模拟电机运行数据，包括正常状态和匝间短路故障状态，通过WebSocket发送到后端服务。

主要功能：

  • 生成三相电流基本波形
  • 模拟匝间短路故障特征：
    • 受影响相的电流幅值增加
    • 相位不平衡
    • 三次谐波增加
    • 负序分量增加
  • 控制故障发生的概率和严重程度
  • 通过WebSocket实时发送数据

技术实现：

  • 使用Python的numpy库生成波形数据
  • 使用websockets库实现WebSocket通信
  • 使用pandas库组织和处理数据

4.2 后端服务

后端服务负责接收实时数据，进行匝间短路故障特征提取和诊断分析，将结果通过WebSocket广播给前端。

主要功能：

  • 接收数据源发送的实时数据
  • 提取匝间短路故障特征：
    • 计算电流不平衡度
    • 计算三次谐波比例
    • 计算负序分量比例
  • 计算综合故障评分
  • 生成诊断结论和建议
  • 将结果广播给所有连接的前端

技术实现：

  • 使用FastAPI框架搭建WebSocket服务
  • 使用scipy库进行信号处理和频谱分析
  • 使用异步编程提高并发处理能力

注意：后端服务需要处理大量实时数据，应优化性能以确保低延迟响应。可以考虑使用批处理机制和数据缓冲策略。

4.3 前端监控界面

前端监控界面负责显示实时电机运行状态、故障指标和诊断结果，提供直观的可视化展示。

主要功能：

  • 连接到后端WebSocket服务
  • 显示实时三相电流波形
  • 显示频谱分析结果
  • 展示故障指标和评分
  • 显示诊断结论和建议

技术实现：

  • 使用Chart.js绘制实时波形和频谱图
  • 使用Element Plus组件库构建UI界面

5. 故障诊断算法

5.1 特征提取

特征 计算方法 故障指示
电流不平衡度 计算三相电流RMS值的最大偏差与平均值的比例 正常值 < 5%，故障时可达 20% 以上
三次谐波比例 计算三次谐波幅值与基波幅值的比例 正常值 < 3%，故障时可达 10% 以上
负序分量比例 计算负序电流分量与正序分量的比例 正常值 < 2%，故障时可达 15% 以上

5.2 故障评分计算

故障评分 = 0.4 * 电流不平衡度 + 0.3 * 三次谐波比例 + 0.3 * 负序分量比例

状态判断:
- 正常: 故障评分 < 0.15
- 预警: 0.15 ≤ 故障评分 < 0.30
- 故障: 故障评分 ≥ 0.30

8. 总结

本方案通过WebSocket实现了匝间短路故障的实时诊断系统，包括数据源模拟、故障特征提取和前端可视化展示。系统具有以下特点：

  • 低延迟架构：数据源直接向后端发送数据，后端处理完成后立即推送到前端，无需中间消息队列
  • 多特征诊断：综合考虑电流不平衡度、三次谐波比例和负序分量比例等多种特征
  • 实时可视化：提供直观的波形图、频谱图和故障指标展示
  • 可扩展性：系统架构支持扩展到其他故障类型和更多连接

通过这种架构，可以实现电机匝间短路故障的早期检测和预警，帮助用户及时采取维护措施，避免设备损坏和生产中断。
